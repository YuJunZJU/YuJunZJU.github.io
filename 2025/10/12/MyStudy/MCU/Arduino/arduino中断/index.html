Arduino中断

发表于2025-10-12|更新于2025-10-12|MyStudyArduino
|浏览量:

中断类型

外部中断

Arduino UNO的外部中断引脚比较少

核心函数

digitalPinToInterrupt(pin):

  • 作用： 这是一个辅助函数，用于将Arduino的引脚号（如 2 或 3）转换为微控制器内部对应的中断号（如 0 或 1）。
  • 为什么不直接用0或1？ 为了代码的可移植性。在不同的Arduino板子上，中断号和引脚号的对应关系可能不同。使用这个函数可以确保你的代码在UNO、Mega等板子上都能正常工作。
  • 示例： 在UNO上，digitalPinToInterrupt(2) 返回 0，digitalPinToInterrupt(3) 返回 1。

ISR :

  • 作用： 这是一个函数名。这个函数就是当中断被触发时，你需要执行的代码块。你必须自己定义这个函数。
  • 注意点：
    • ISR应该尽可能快地执行完毕
    • 禁止使用delay()，因为delay()依赖于定时器，而中断会影响定时器的正常工作
    • 谨慎使用串口，因为串口通信本身也依赖于中断
    • 共享变量必须声明为 volatile: 如果一个变量在ISR中被修改，并且在loop()或其他函数中也会被使用，那么在定义它时必须加上volatile关键字。

mode (触发模式):

  • 作用： 定义了什么样的电平变化会触发中断。有四种模式可选：
    • LOW: 当引脚为低电平时，持续触发中断。（慎用，容易导致程序卡在中断里）
    • CHANGE: 当引脚电平发生任何改变时（从高到低 或 从低到高）触发。
    • RISING: 当引脚电平从低电平变为高电平时（上升沿）触发。
    • FALLING: 当引脚电平从高电平变为低电平时（下降沿）触发。对于按钮按下（使用上拉电阻）的场景，FALLING 是最常用的模式。

volatile的作用

现代的编译器非常智能，为了让你的程序运行得更快、占用内存更少，它会进行大量的优化。

其中一种常见的优化是：如果编译器发现在一段代码里，一个变量被读取了多次，但期间没有代码去修改它，编译器就会觉得“何必每次都去内存里读呢？我把它读一次，存到CPU的一个临时仓库（寄存器, Register）里，后面直接从这个飞快的寄存器里取值就行了。”

我们来设想一下如果上面的代码中没有volatile会发生什么（假设loop中有if (ledState == true)：

  1. 编译器开始分析 loop() 函数。

  2. 它看到了第一次 if (ledState == true)。它从主内存（RAM）中读取 ledState 的值（初始为 LOW/false），并可能把它缓存到一个CPU寄存器里。

  3. 它继续往下分析 loop() 里的其他代码。在它看来，loop() 函数内部没有任何代码会去修改 ledState 的值。

  4. 当它看到第二次 if (ledState == true) 时，它会想：“我刚刚已经读过 ledState 了，而且中间没有任何代码修改它，我没必要再去内存里慢悠悠地读一次了。直接用我缓存 G 在寄存器里的旧值（false）就行！”

  5. 编译器完全不知道 myISR() 函数的存在，也不知道它会在任何时刻突然闯进来，修改了内存里 ledState 的真实值。

volatile 的本质： 告诉编译器不要对该变量的访问进行优化

引脚变化中断

支持的引脚比较多， (UNO上几乎所有数字和模拟引脚都支持)

PCINT(引脚变化中断功能)

在ATmega328P芯片（Arduino UNO的核心）上，支持PCINT的引脚被分成了三个组：

  • PCINT0 (组0): 对应Arduino引脚 D8, D9, D10, D11, D12, D13
  • PCINT1 (组1): 对应Arduino引脚 A0, A1, A2, A3, A4, A5
  • PCINT2 (组2): 对应Arduino引脚 D0, D1, D2, D3, D4, D5, D6, D7

上述不一定对（是AI说的，具体应该要查手册）

关键点：

  • 每个组共享一个中断。
  • 只要组内任何一个被启用的引脚电平发生变化，就会触发该组的中断。
  • 你需要为每个启用的组编写一个对应的中断服务程序（ISR）。

3. 如何设置PCINT？—— 操作寄存器

设置PCINT不像attachInterrupt那样只有一个函数，你需要手动配置三个关键的寄存器。

  1. PCICR - 引脚变化中断控制寄存器
    • 这个寄存器用来开启或关闭整个组的中断功能。
    • 它有三个关键位：PCIE0, PCIE1, PCIE2。
    • 要启用组0的中断，你就需要把PCIE0位设为1。
    • 代码示例 (启用组0): PCICR |= bit(PCIE0); (或者 PCICR |= B00000001;)
  2. PCMSKx - 引脚变化屏蔽寄存器
    • PCICR打开了组的大门后，PCMSKx（PCMSK0, PCMSK1, PCMSK2）用来选择你关心组内的哪几个具体引脚。
    • PCMSK0 对应组0，PCMSK1 对应组1，以此类推。
    • 每个寄存器有8个位，每一位对应组内的一个引脚。将某位设为1，就表示启用该引脚的中断。
    • 代码示例 (在组0中，启用D10引脚的中断):
      • 查表可知，D10对应的是PCINT2。
      • PCMSK0 |= bit(PCINT2); (或者 PCMSK0 |= B00000100;)
  3. ISR(PCINTx_vect) - 中断服务程序 (The Handler)
    • 这是中断触发后实际执行的函数。
    • 组0的中断向量是 PCINT0_vect，组1是 PCINT1_vect，组2是 PCINT2_vect。

总结一下，如果你想开启某个引脚的引脚变化中断的话，流程是

  1. 打开整组的开关
  2. 打开相应引脚的开关
  3. 编写中断函数逻辑（其中要自己判断是上升沿还是下降沿）
